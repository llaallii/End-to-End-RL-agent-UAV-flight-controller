Test document for development of Doorstop

1.1 FW001

The selected MCU shall have a CPU clock >=168 MHz, RAM >=256 KB, Flash >=512 KB, and a hardware FPU.

Parent links: SYS001

1.1 FW010

The HIL communication protocol shall use a simple binary encoding with error detection (e.g., CRC) over UART or USB.

1.2 FW002

The firmware shall run on a real-time operating system (RTOS), with FreeRTOS as the recommended choice.

1.3 FW003

The firmware shall be organized into isolated modules: sensor, control, communication, and safety.

1.4 FW004

The firmware shall use static memory allocation for all real-time tasks; dynamic heap usage shall be minimized.

1.5 FW005

RL policy inference (forward pass) shall complete in <5 ms on the target MCU using INT8 quantization, targeting ~3 ms.

1.6 FW006

The complete control loop (sensor read, inference, motor output) shall execute in <10 ms (>=100 Hz).

1.7 FW007

Control loop timing jitter shall be <1 ms.

1.8 FW008

End-to-end latency from sensor-to-actuator shall be quantified, documented, and acceptable for stable flight.

1.9 FW009

The embedded RL inference output shall match the PC-based FP32 inference output within a documented acceptable tolerance.

1.11 FW011

The HIL interface shall support bidirectional exchange of sensor data (Gazebo to MCU) and control commands (MCU to Gazebo).

1.12 FW012

The firmware communication driver shall implement buffering, timeout handling, and error recovery.
